## Source 2
**MDN Web Docs, Array.prototype.some()**
https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Global_Objects/Array/some

## Keywords I searched with
some() method in javaScript 

## Summary 
ეს მეთოდი ახალი აღმოჩნდა ამიტომ გარდა AI-ის ახსნისა, ინტერნეტ რესურსებშიც წავიკითხე მისი აღწერა, some() არის მასივის ჩაშენებული მეთოდი რომელიც აბრუნებს boolean-ს. 
აბრუნებს true-ს თუ პოულობს იმ ელემენტს, რომელიც აკმაყოფილებს ფუნქციის პირობას, სხვა შემთხვევაში false-ს. some() მეთოდი არ ცვლის იმ მასივს, რომელზეც ის გამოიძახება, თუმცა callbackFn-ის სახით გადაცემულმა ფუნქციამ შესაძლოა ის შეცვალოს. 
ყველაზე მნიშვნელოვანი იყო ის შედარება თუ რატომ ჯობს ეს მეთოდი სხვა მეთოდებს, მაგალითად filter()-ს ან forEach()-ს. ის პირობითი შემოწმებისთვის უფრო ეფექტურია,რადგან მუშაობას წყვეტს პირველივე შესაბამისობის აღმოჩენისთანავე და პირდაპირ boolean მნიშვნელობას აბრუნებს. filter()-ისგან განსხვავებით, არ ქმნის ახალ მასივს და არ გადის იმ ელემენტებზე, რომელთა შემოწმებაც არ გვჭირდება. forEach()-ს არ შეუძლია მუშაობის ნაადრევად შეწყვეტა და არაფერს აბრუნებს (undefined), რაც ნიშნავს, რომ ციკლის შეწყვეტის ლოგიკა დამოუკიდებლად უნდა დაიწეროს. find() თავად ელემენტს აბრუნებს, მაშინ როცა some() უფრო უსაფრთხო და მარტივია, თუ მხოლოდ true/false შემოწმება გვინდა. 

## Source 3
**MDN Web Docs, Window: localStorage property**
https://developer.mozilla.org/en-US/docs/Web/API/Window/localStorage

## Keywords I searched with
localStorage, getItem/setItem 

## Summary 
უფრო ვრცლად გავეცანი, მაგალითებით. localStorage-ში შენახული მონაცემები ინახება ბრაუზერის სესიების განმავლობაში. 
getItem() მეთოდი, რომელსაც არგუმენტად გადაეცემა გასაღების სახელი, აბრუნებს ამ გასაღების მნიშვნელობას (ან null-ს, თუ გასაღები არ არსებობს) მოცემულ Storage ობიექტში.
setItem() მეთოდი, რომელსაც არგუმენტებად გადაეცემა გასაღების სახელი და მნიშვნელობა, ამატებს აღნიშნულ გასაღებს მოცემულ Storage ობიექტში ან ანახლებს მის მნიშვნელობას, თუ გასაღები უკვე არსებობს.
